\begin{abstract}
Ensuring reliability in modern digital systems requires both effective fault masking and efficient resource utilization. 
Conventional redundancy techniques such as Triple Modular Redundancy (TMR) provide strong fault masking but incur constant area and power overhead, while algorithm-based techniques such as residue checking enable fault detection but cannot prevent error propagation.

This paper proposes a hybrid self-healing digital architecture that integrates adaptive modular redundancy with residue-based fault localization. 
The architecture dynamically adjusts redundancy levels (single, TMR, or PMR) based on runtime fault conditions, while a lightweight modulo-3 residue checker enables per-module fault detection and selective quarantine without duplicating full datapaths.

The proposed design achieves both fault masking and fault detection within a unified framework. 
Monte Carlo fault injection across 1–50\% error probability demonstrates detection rates up to 86\% with silent data corruption below 1\%. 
Compared to static TMR, the proposed architecture improves detection capability while maintaining comparable reliability and controlled hardware overhead. 
FPGA synthesis on an Artix-7 device shows modest resource overhead (173 LUTs / 44 FFs), demonstrating practical feasibility.

\end{abstract}

\begin{IEEEkeywords}
Adaptive Redundancy, Residue Verification, Fault Localization, TMR, PMR, FPGA, Hardware Self-Healing
\end{IEEEkeywords}

\section{Introduction}
\label{sec:intro}

\subsection{Problem Statement}

Hardware reliability is critical in radiation-prone environments (aerospace, deep-space) and safety-critical systems 
(autonomous vehicles, medical devices, nuclear control).
Single-event upsets (SEUs), aging-related failures, and electromagnetic interference cause bit-flips and stuck-at faults 
that can propagate to system failure if undetected or unmasked~\cite{nasa_fault,av_fault}.

Triple Modular Redundancy (TMR)~\cite{tmr_classic} is the industry standard solution: replicate hardware three times and use 
majority voting to mask any single fault. However, TMR operates at constant 3$\times$ area and power overhead \textit{regardless of actual risk level}.
This is wasteful: in low-risk operational periods (nominal conditions, benign thermal environment), the redundancy provides no additional benefit.

\subsection{Research Gap}

Existing fault-tolerant architectures exhibit fundamental limitations. 
Static redundancy schemes such as TMR provide fault masking but incur constant overhead and lack adaptability to runtime fault conditions. 
Algorithm-based techniques such as residue or checksum verification enable fault detection but cannot prevent erroneous outputs without recomputation. 
Adaptive redundancy approaches improve efficiency but typically lack fine-grained fault localization and recovery mechanisms.

Therefore, there is a need for an architecture that simultaneously provides:
(i) dynamic redundancy adaptation, 
(ii) fault masking capability, and 
(iii) fine-grained fault detection and localization.

This work addresses this gap by integrating adaptive modular redundancy with residue-based fault localization and selective quarantine.

\subsection{Contributions}

The key contributions of this work are:

\begin{itemize}
\item A hybrid adaptive redundancy architecture that dynamically switches between single, TMR, and PMR modes based on runtime fault conditions.
\item A lightweight modulo-3 residue-based fault localization mechanism enabling per-module error detection without duplicating full datapaths.
\item A selective quarantine policy that isolates faulty modules to improve reliability under multi-fault scenarios.
\item A comprehensive evaluation using Monte Carlo fault injection, demonstrating improved detection capability and reduced silent data corruption compared to static redundancy.
\end{itemize}

\section{Evaluation Metrics}

The following metrics are used to evaluate system reliability:

\textbf{Detection Rate:}
\begin{equation}
D = \frac{\text{Detected Faults}}{\text{Injected Faults}}
\end{equation}

\textbf{Masking Rate:}
\begin{equation}
M = \frac{\text{Correct Outputs under Fault}}{\text{Injected Faults}}
\end{equation}

\textbf{Silent Data Corruption (SDC):}
\begin{equation}
SDC = \frac{\text{Undetected Faults}}{\text{Injected Faults}}
\end{equation}

\textbf{Accuracy:}
\begin{equation}
Accuracy = \frac{\text{Correct Outputs}}{\text{Total Operations}}
\end{equation}

\subsubsection{Mathematical Foundation}

For integer modular arithmetic, a residue check detects single-bit errors with probability proportional to the modulus $M$.
For $M = 3$ (chosen for hardware efficiency), the detection coverage is:

\begin{equation}
  P(\text{detect} \mid \text{unidirectional error}) = 1 - \frac{1}{M} = \frac{2}{3} \approx 67\%
  \label{eq:mod3_coverage}
\end{equation}

For truly random bit-flips (Hamming weight $w$ bits flipped in $n$-bit word), let
$P_0(w)$ denote the probability that the error sum is $0 \pmod{3}$.
Then detection increases as:

\begin{equation}
\begin{aligned}
  P(\text{detect} \mid w)
  &= 1 - P_0(w) \\
  &\approx 1 - \frac{1}{3^w}
\end{aligned}
  \label{eq:random_flip_coverage}
\end{equation}

For single bit-flip ($w=1$): $P(\text{detect}) = 2/3 \approx 67\%$.
For two bit-flips ($w=2$): $P(\text{detect}) \approx 8/9 \approx 89\%$.

This justifies mod-3 over smaller moduli (e.g., mod-2 parity with only 50\% coverage) 
and over larger moduli (mod-7 adds marginal coverage gain at 2$\times$ gate count).

\subsubsection{Hardware Implementation}

Each residue checker consists of:

\begin{enumerate}
  \item \textbf{Digit-Sum Mod-3 Reducer:} Computes $r = x \bmod 3$ via recursive digit-sum reduction (3 stages for 8-bit input).
        Gate count: 12--16 LUTs.
  
  \item \textbf{Expected Value Predictor:} For ADD: $r_{\text{exp}} = (r_a + r_b) \bmod 3$. 
        For SUB: $r_{\text{exp}} = (r_a - r_b) \bmod 3$. Non-arithmetic ops skip validation.
        Gate count: 4--8 LUTs.
  
  \item \textbf{Comparison Logic:} Assert \texttt{mismatch} if $r_{\text{result}} \ne r_{\text{exp}}$ and \texttt{valid}=1.
        Gate count: 2 LUTs.
\end{enumerate}

\textbf{Total per checker: 16--20 LUTs, 0 FFs.}
Compared to external residue checkers (50+ LUTs) or Hamming-code ECC modules (80+ LUTs), mod-3 is 2--5$\times$ more area-efficient.

\begin{algorithm}[t]
\caption{Modulo-3 Residue Verification (Per ALU)}
\label{alg:residue_verify}
\footnotesize
\begin{algorithmic}[1]
\Require $a,b,\texttt{op},y$
\Ensure $\texttt{valid},\texttt{mismatch}$
\State $r_a \gets a \bmod 3$, $r_b \gets b \bmod 3$
\State $r_y \gets y \bmod 3$
\If{$\texttt{op} = \texttt{ADD}$}
  \State $r_{exp} \gets (r_a + r_b) \bmod 3$; $\texttt{valid} \gets 1$
\ElsIf{$\texttt{op} = \texttt{SUB}$}
  \State $r_{exp} \gets (r_a - r_b) \bmod 3$; $\texttt{valid} \gets 1$
\Else
  \State $r_{exp} \gets 0$; $\texttt{valid} \gets 0$
\EndIf
\State $\texttt{mismatch} \gets \texttt{valid} \land (r_y \neq r_{exp})$
\end{algorithmic}
\end{algorithm}

\subsection{Hybrid Quarantine Policy}

Figure~\ref{fig:quarantine_fsm} shows the per-module quarantine state machine and event fusion logic.

\begin{figure*}[htbp]
\centering
\includegraphics[width=0.68\textwidth]{../architectures/figure3.png}
\caption{Quarantine Controller State Machine: Per-module active/quarantine state tracking, event fusion logic (configurable policy), trip and clear counter thresholds.}
\label{fig:quarantine_fsm}
\end{figure*}

The quarantine controller fuses \textit{residue events} (detected arithmetic anomalies) and \textit{voter events} (output disagreement). 
Four configurable policies:

\begin{enumerate}
  \item \textbf{Residue-Only:} Isolate if $N$ consecutive \texttt{mismatch}=1 from residue checker.
  \item \textbf{Voter-Only:} Isolate if $N$ consecutive voter disagreement detected.
  \item \textbf{Fused-AND:} Isolate only if both residue mismatch AND voter disagreement occur in same cycle for $N$ consecutive cycles.
  \item \textbf{Fused-OR:} Isolate if either event occurs continuously for $N$ cycles.
\end{enumerate}

Once isolated, a module is tracked by a \textit{good-streak counter}. 
After $M$ consecutive fault-free cycles, the module is reintegrated into voting.
This provides \textit{transient fault resilience}: temporary upsets do not permanently disable a module.

Default tuning: $\texttt{QUAR\_TRIP}=3, \texttt{QUAR\_CLEAR}=10$. 
(Results are robust across thresholds 2--5 for trip and 8--15 for clear.)

\begin{algorithm}[t]
\caption{Per-Module Quarantine State Policy}
\label{alg:quarantine_policy}
\footnotesize
\begin{algorithmic}[1]
\Require $r_{evt}$, $v_{evt}$, policy $P$
\Ensure module state $\in \{\texttt{ACTIVE},\texttt{QUARANTINED}\}$
\State $e \gets \textsc{FuseEvent}(P,r_{evt},v_{evt})$
\If{state $= \texttt{ACTIVE}$}
  \If{$e = 1$}
    \State $\texttt{trip\_count} \gets \texttt{trip\_count}+1$
    \State $\texttt{good\_count} \gets 0$
  \Else
    \State $\texttt{trip\_count} \gets 0$
  \EndIf
  \State $h \gets$ number of healthy modules
  \If{$\texttt{trip\_count} \ge \texttt{QUAR\_TRIP}$ and $h \ge 3$}
    \State state $\gets \texttt{QUARANTINED}$; $\texttt{trip\_count} \gets 0$
  \EndIf
\Else
  \If{$e = 0$}
    \State $\texttt{good\_count} \gets \texttt{good\_count}+1$
  \Else
    \State $\texttt{good\_count} \gets 0$
  \EndIf
  \If{$\texttt{good\_count} \ge \texttt{QUAR\_CLEAR}$}
    \State state $\gets \texttt{ACTIVE}$; $\texttt{good\_count} \gets 0$
  \EndIf
\EndIf
\Statex
\Function{FuseEvent}{$P,r,v$}
  \If{$P=0$} \State \Return $r$ \Comment Residue-Only
  \ElsIf{$P=1$} \State \Return $v$ \Comment Voter-Only
  \ElsIf{$P=2$} \State \Return $r \land v$ \Comment Fused-AND
  \Else \State \Return $r \lor v$ \Comment Fused-OR
  \EndIf
\EndFunction
\end{algorithmic}
\end{algorithm}

\subsection{Risk Estimation and Mode Selection}

The mapping below applies to the \textbf{Adaptive (+Residue +Quarantine)} architecture.

The risk estimator samples two channels:
\begin{itemize}
  \item \textbf{Switching Activity:} Toggle count on primary ALU output per \texttt{WINDOW\_CYCLES}.
  \item \textbf{Error Frequency:} Count of voter mismatch pulses per window.
\end{itemize}

Each channel independently classified as LOW/MEDIUM/HIGH based on thresholds 
($\texttt{TOGGLE\_MED}=10, \texttt{TOGGLE\_HIGH}=30, \texttt{ERROR\_MED}=2, \texttt{ERROR\_HIGH}=4$).
The final risk = \textit{max}(activity risk, error risk) at end of window.

\subsection{Residue-Based Fault Localization Improvements}

The residue checker provides \textit{per-module} detection signal without requiring external redundancy. 
This enables selective module quarantine, offering three tangible improvements:

\begin{enumerate}
  \item \textbf{Increased Detection Granularity:} Unlike bulk voter disagreement (which detects `some module is bad' but not which), 
        residue checkers pinpoint faulty module to arithmetic operation type and module instance.
  
  \item \textbf{Transient Fault Resilience:} By quarantining a module after $N$ consecutive errors and reintegrating after $M$ good cycles, 
        single-cycle upsets do not permanently degrade system robustness.
  
  \item \textbf{Reduced Masking Time:} In Adaptive TMR, once a module is identified as faulty and quarantined, the controller 
        selects a non-faulty replacement for the voter, improving masked-fault recovery time by 1--2 cycles versus static TMR.
\end{enumerate}

\textbf{Quantitative Impact:} In two-fault scenarios S8, S9, and S13, Static TMR drops to 0\% while
Adaptive (+R+Q) sustains 100\% by quarantining faulty modules and maintaining correct PMR voting among healthy instances.

\section{Conclusion and Future Work}

This paper presented a hybrid adaptive fault-tolerant architecture that integrates modular redundancy with residue-based fault localization. 
The proposed design achieves both fault masking and detection while dynamically adapting redundancy levels to runtime conditions.

Experimental results demonstrate improved reliability and detection capability compared to static TMR, with minimal increase in hardware overhead. 
The architecture is particularly effective in reducing silent data corruption under high fault rates.

Future work includes extending the approach to multi-modulus residue systems, exploring machine learning-based fault prediction, and scaling the architecture to larger datapaths and system-level designs.
